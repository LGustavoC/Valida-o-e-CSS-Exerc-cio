Letra A:

É possível sobrepor à aparência (estilo) definida em regras CSS anteriores, mais genéricas, por regras mais específicas? Se possível, essa técnica tem nome? Exemplifique.

Sim, se chama overriding. Por exemplo, um style que foi definido para 3 tags (h1,h2,h3), se fizer um style mais específico (para a tag h3 por exemplo) ele ficará com ele.

Letra B:

Caso definíssemos uma regra CSS mais geral para cabeçalhos de nível 1, 2 e 3, seguida de outra somente para o nível 1, como seria o comportamento do navegador? Descreva os vários cenários possíveis e o comportamento do navegador em cada uma delas.

Como dito na letra A, um style mais específico será sempre usado pelo navegador em vez de um mais geral.

Letra C:

É sempre interessante otimizar o código CSS? Cite algumas vantagens.

Sempre é melhor otimizar pois evita duplicação de código, dando assim menos trabalho para o navegador que ficaria lendo informação repetida.

Letra D:

Defina herança em CSS. Utilize, também, imagens para ilustrar tais definições.

Basicamente significa que os filhos herdam de seus pais ,ou seja, se eu definir uma característica para um elemento que é mais geral (por exemplo body) as tags filhas vão herdar aquelas características (podendo ser alteradas por argumentos mais específicos).

Letra E:

Podemos sobrescrever características herdadas todas as vezes que desejarmos deixá-la mais específica?

Como dito na questão D, sim, pois os argumentos mais específicos são sempre levados mais em conta.
